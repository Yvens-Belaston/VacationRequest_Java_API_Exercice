Auth model. Bonita uses a session cookie. Call login first; the response sets a JSESSIONID cookie plus an X-Bonita-API-Token. Send the cookie on every later request, and echo the token in an X-Bonita-API-Token header for any POST/PUT/DELETE. The original Java also adds HTTP Basic auth (http-api / h11p-@p1) at the reverse-proxy layer — keep that header if your server requires it.
